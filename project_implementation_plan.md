# 穹盾智矿 —— 露井联采空天地一体化智能预警系统 实施落地计划 

## 阶段零：项目启动与环境基建 (第 1-2 周)
**目标：搭好台子，明确技术选型，让各小组能开始写代码。**

*   **服务器与中间件部署：**
    *   在开发服务器上使用 Docker 部署核心组件：MySQL (存业务数据)、Redis (缓存和消息队列)、InfluxDB/TDengine (存传感器时序数据)、EMQX (作为物联网 MQTT Broker)。
*   **代码仓库与 CI/CD 构建：**
    *   建立 GitLab/GitHub 仓库，分为 `backend` (后端)、`frontend` (Web端和大屏)、`ai` (算法服务) 三个核心目录。
    *   配置好基础的自动化测试与部署流水线。
*   **Mock 数据生成器开发：**
    *   编写一个简单的 Python 脚本，模拟传感器阵列，通过 MQTT 协议不断向 EMQX 发送假数据（如模拟微震和裂缝宽度的变化），用于跑通整个管道流程。

## 阶段一：数据底座与核心管理平台 MVP (第 3-6 周)
**目标：实现数据的“传得上来、存得下去、看得到图表”，打通数据闭环。**

*   **物联网网关与数据消费流 (Backend)：**
    *   开发后端微服务 (推荐 Go 或 Java Spring Boot)，订阅 EMQX，对接收到的数据进行格式校验、异常值清洗。
    *   将清洗后的数据持久化写入 InfluxDB (时序数据) 和 MySQL (设备最后状态)。
*   **核心业务后台平台开发 (Frontend & Backend)：**
    *   **设备资产平台：** 实现设备增删改查 (CRUD)、设备类型定义、绑定的经纬度空间属性等。
    *   **RBAC 权限系统：** 用户登录、角色管理。
    *   **基础告警规则引擎：** 开发简单的阈值告警功能（如位移 > 10cm 则触发），并在 MySQL 中生成告警记录。
*   **初步的可视化展示：**
    *   利用 Vue3 + ECharts，通过接口调取 InfluxDB 数据，展示传感器时序折线图（例如，显示某裂缝过去 24 小时的宽度变化趋势）。

## 阶段二：深入骨髓——AI 算法封装与智能辅助 (第 7-12 周)
**目标：将计划书中的高难度 AI 算法实装，替代阶段一的简单规则告警。**

*   **YOLOv8-Mine 视觉算法工程化 (AI)：**
    *   收集矿区裂缝、渗水等数据集进行标注。
    *   训练 ESRGAN-Lite 超分辨率模型和 YOLOv8 检测模型。
    *   将模型封装为 FastAPI 接口，实现接收图片/视频流 -> 超分预处理 -> 缺陷检测 -> 输出含坐标和置信度的 JSON 的完整链路。
*   **多源融合时序预测模型实装 (AI)：**
    *   基于收集的历史矿区坍塌或位移数据，训练所涉及的预测模型 (如 `XGBoost`, `LSTM`, 或计划书中提到的 `HEOA-XGBoost` 等)。
    *   同样封装为微服务，定时去拉取近期的 InfluxDB 数据，进行趋势推演，并预测未来 1-3 天是否超限。
*   **三级阶梯式智能预警模块构建 (Backend)：**
    *   接入 AI 预测接口，搭建监控级(轻微超限)、预警级(局部异常/多因子异常)、警报级(剧烈变化)判断逻辑。
    *   打通多渠道推送系统：系统内站内信、生成事件工单流程，并通过对接第三方 API 实现短信、App 推送。

## 阶段三：面子工程——三维数字孪生大屏开发 (第 13-18 周)
**目标：实现震撼的可视化交互体验，在数字世界 1:1 复刻矿山并在其上动态映射风险。**

*   **矿区 3D 资产入库与切片处理：**
    *   将无人机航拍倾斜摄影模型、激光雷达点云等数据，利用工具（如 Cesium ion 或 osg23dtiles）转换为 `3D Tiles` 格式，便于 Web 端流畅加载。
*   **三维数字底盘研发 (Frontend)：**
    *   采用 `Vue3 + Cesium.js` 或 `Three.js`，加载地球图层与 3D Tiles 矿山模型。
    *   实现模型上的漫游、视角切换等基础交互操作功能。
*   **动态数据 1:1 映射与渲染 (Frontend)：**
    *   通过 WebSocket 与后台建立长连接。
    *   在地图对应物理坐标上渲染代表传感器的虚拟空间节点（如气泡、指示针）。
    *   一旦后端推送某节点告警状态（红/橙/黄/绿），立刻改变该空间节点的颜色，甚至在点击时弹出实时视频流或 AI 分析曲线悬浮窗。

## 阶段四：空地协同——无人机调度与系统闭环 (第 19-24 周)
**目标：实现机器代替人工的终极自动化，让告警触发无人机去现场“查证”。**

*   **无人机地面监控站对接 (Backend & AI)：**
    *   通过无人机厂商提供的 SDK (如大疆 MSDK/Cloud API)，实现云端对其基本指令如起飞、按航线飞行的控制。
*   **自动化工单及调度协同闭环：**
    *   编写逻辑：当三级预警触发且该区域缺乏清晰视野时系统自动生成任务，基于环境路况下发给对应的无人机。
    *   融合此前研发的无人机多机避障和路径规划算法（基于实时传感器数据微调路径）。
*   **系统压测、联调与交付：**
    *   进行大规模并发测试：用脚本模拟 10,000+ 个传感器的高频上报，测试系统瓶颈及数据吞吐量。
    *   进行真实露井联采矿山的场景试运行（先小规模试水，如在一个局部边坡部署验证所有链路）。
    *   交付验收，整理运维文档册、培训大纲。
