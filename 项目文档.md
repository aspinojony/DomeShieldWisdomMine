# 穹盾智矿 (DomeShield Wisdom Mine) - 核心技术与业务文档

## 一、 项目概述简介

**定位**：露井联采空天地一体化智能预警中枢系统 (Sky-Earth Integrated Intelligent Warning Center for Open-Pit and Underground Mining)。

本项目是一个综合、复杂且深入应用现代 Web 与 AI 技术的矿山安全监控和数字孪生平台。系统利用**多模态感知数据**（地表裂缝、微震监测、沉降、设备倾角等物联网边缘传感器）、**无人机(UAV)空地协同视觉巡检**等手段，搭配**时序预测模型(LSTM)**以及\**计算机视觉识别引擎(YOLO)*8，实现对露天矿坑及地下巷道的高效监测与预测警告，并配合一套完善的后台业务管理及生产调度系统。

---

## 二、 系统架构图解

系统遵循**边缘感知—>数据高速总线—>AI研判/业务处理—>3D大屏展现**的分层微服务架构体系，各功能互相解耦。

### 1. 结构划分

- **传感器边缘层 (IoT/Edge)**: `mock_devices/` - 包含大量硬件模拟器产生实时物联网传感报文。
- **消息总线与中间件层 (Message Bus)**: 依赖 `EMQX(MQTT)` 网关完成感知设备海量数据接入，通过 `Apache Kafka` 提供高吞吐的流计算平台支撑。
- **AI 与预测引擎层 (AI Engine)**: `ai_core/` - 提供纯算力层的推理/训练服务（视觉侦察服务 + 时序预警服务）。
- **核心业务平台层 (Backend)**: `backend_service/` - FastAPI, 提供 REST 接口控制、工单流转、权限验证、数据清洗下沉（InfluxDB）功能。
- **交互与展现层 (Frontend)**: `frontend_dashboard/` - Vue3 + 三维孪生引擎(Cesium/Three.js)，提供“总览大屏”、“视频监控”、“生产运营展示”、“智能AI识别面板”及成熟的“Admin后台管理”。

## 三、 核心技术栈概览

1. **前端 (Frontend)**
   - 框架：`Vue 3` + `Vite` + `Vue Router 4`
   - UI 风格：暗黑工业科幻风 (Cyber-industrial Glassmorphism)
   - 数据可视化：`ECharts` (支持复杂组合式统计图)、`Three.js` / `Cesium` (3D 矿山数字孪生引擎)
   - 网络请求：`Axios`

2. **后端业务与网关 (Backend Service)**
   - 框架：`FastAPI` (Python 3.10+)
   - 认证鉴权：`JWT (Jose)`, `passlib(bcrypt)` 密码加盐哈希
   - 关系型数据库管理：`SQLAlchemy` + `SQLite` (可轻松迁移至 MySQL/PostgreSQL)
   - 时序数据库：`InfluxDB 2.0` (存储连续的工业物理时序波形，以提供前台回填展示)
   - 数据流摄取管道：`paho-mqtt`, `kafka-python`

3. **人工智能与视觉引擎 (AI Core)**
   - 模型框架：`PyTorch`
   - 时序风险预测：**MultiSensorLSTM 网络** (多模态融合特征的二分类及风险评分预测)
   - 计算机视觉：**YOLOv8** (用于无人机拍摄图像的实时裂缝检测、长宽测算及告警评级)

## 四、 核心系统与功能模块剖析

### 1. 系统角色与安全架构 (RBAC)

- 平台采用强隔离式角色访问控制（RBAC）：
  - **系统管理员 (Admin)**：最高权限，可增删改查设备、规则、注册新账号。
  - **工程师 (Engineer)**：负责任务接报、执行告警确认(ACK)闭环、有权控制和调度无人机(UAV)下发航线任务。
  - **观察员 (Viewer)**：只读访问大屏各项数据。

### 2. 边缘端数据接揽与流处理 (Data Ingestion & Stream)

- **MQTT 数据桥接 (`data_ingestion.py`)**：快速拉取边缘传感数据，内置安全红线熔断器，识别到严重数值偏差立刻触发一级预警，并将数据点以纳秒精度下沉写入 InfluxDB。
- **Kafka 吞吐消费 (`kafka_stream_processor.py`)**：为海量高频的声发射和微震波束提供高吞吐的消费能力，结合业务支持滑动窗口式的限流与二次告警计算。

### 3. 多模态 AI 推演引擎 (`ai_prediction_engine.py`)

- 利用加载权重后的 `MultiSensorLSTM` 实时抽血当前矿山 60 个步长窗口（包括裂缝、倾角、微震能量、沉降）的值进行特征前馈推理。
- 当探知地质结构有塌方风险阈值（>85%概率）时，引擎独立剥离出**自动派单权**，不经人工同意强制触发空地协同——向系统中待命状态的无人机(`UAV-EAGLE`)生成避障航线并勒令起飞。

### 4. UAV 空地协同视觉中枢 (`vision_inference_service.py`)

- 提供无人机图像回传 AI 分析的 HTTP 网关 (`:8003/api/v1/vision/analyze_crack`)。
- 当无人机经前文的联动抵达指定空间坐标点后，拍摄高清现场图传入视觉接口，启动 YOLOv8 检测识别出地表张拉裂缝的像素大小、数量与置信度。
- 将识别物物理换算 (Pixel to mm)，当裂纹物理宽度超标时，作为确认告警（不可篡改痕迹）返给指挥中心并形成永久闭环。

### 5. 业务后台平台能力 (Admin Center)

提供可操作化后台：

- **设备台账**：矿山物联网智能装备的电子户口本CRUD。
- **告警规则引擎**：支持给不同设备灵活配置基于阈值(`>` `<` `==`)和度量字段 (`metric_field`) 的预警包（例：`crack_width_mm` > 10.0 设置为警报级）。
- **告警处理生命周期**：所有的违规越界记录录入警报墙，必须由工程师或管理员排查后手签 `ACK` (处理确认) 才能销案，保证隐患清零。

### 6. 日常生产运营调度 (Production Ops)

涵盖：产量KPI、燃油消耗对比、车辆效率 OEE、爆破物质成本、安全态势综合统计等；此外引入基于车流量和工作面分布的**装卸卡口拥堵监控**引擎。

## 五、 本地运行环境配置与启动指南 (Quick Setup)

项目中各个核心子系统已被解耦在不同端口独立交互，可按需启动构建完整的“空天地一体孪生节点”：

### 第 1 步：基础设施挂载

需拉起包括 `MQTT (EMQX/Mosquitto)`、`Kafka`、`Zookeeper` 与 `InfluxDB 2.0` 在内的底层服务组件集群。

> (可通过 `ops/docker-compose.yml` 统一拉起)

### 第 2 步：运行 Backend 服务网关

> **端口：8002** (API 服务) / **8000** (快速查询服务) / 同时后台常驻数据收集网关。

```bash
cd backend_service
source venv/bin/activate
pip install -r requirements.txt
python data_ingestion.py &                           # 拉起 MQTT 时序存储网关
uvicorn business_api:app --host 0.0.0.0 --port 8002  # 拉起核心鉴权/业务网关
uvicorn api_server:app --host 0.0.0.0 --port 8000    # 拉起基础传感器历史报文网关
```

### 第 3 步：运行 AI 引擎 (时空LSTM和边缘视觉分析)

> **端口：8001** (时序LSTM中枢) / **8003** (YOLO视觉中枢)

```bash
cd backend_service
python ai_prediction_engine.py &      # 拉起多模态 AI 时空推演调度引擎 (Port 8001)

cd ../ai_core
python vision_inference_service.py &  # 拉起无人机视觉中枢 (Port 8003)
```

### 第 4 步：运行 Frontend 大屏

> **端口：5173** (前端Vite Dev)

```bash
cd frontend_dashboard
npm install
npm run dev
```

### 第 5 步：唤起边缘数据发生器（Edge Mocks）

启动分布式的终端注入脚本以观察前端告警：

```bash
python mock_devices/iot_sensor_simulator.py
```

## 六、 未来规划与进阶架构展望 (Roadmap)

1. **三维真实 GIS 地形切片集成**：目前依赖于基础的 `Three.js` 或随机地形体系，计划更深度集成 `Cesium` 与 `OSGB` 或 `3DTiles` 进行一比一实景贴图建模。
2. **多云调度与边缘 K3s**：重构 AI 推理模块引擎到更轻量级的 `NCNN` 架构下发至现场树莓派/Jetson等边缘盒子集群侧运算，实现即时断网闭环。
3. **Flink 流拓扑图引入**：针对 Kafka 中繁杂的声发射信道波形数据，引入分布式的流窗口计算图计算能量峰值。
